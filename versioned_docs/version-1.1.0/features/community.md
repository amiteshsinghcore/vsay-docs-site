# Community (Issue Tracker)

WebXTerm includes a built-in **Community** feature — a collaborative issue tracker for your infrastructure team. Think of it as an internal knowledge base and ticketing system where team members can report problems, share solutions, and track infrastructure issues without leaving the platform.

## Overview

The Community feature runs as a separate backend service and provides:

- **Issue creation** — Report infrastructure problems, incidents, or questions
- **Issue tracking** — Track open, in-progress, and resolved issues
- **Threaded discussions** — Comment and collaborate on issues with your team
- **Voting** — Upvote issues that affect multiple team members
- **Search** — Find known issues and solutions quickly

## Accessing Community

The Community section is available from the main WebXTerm dashboard:

1. Log in to your WebXTerm dashboard
2. Click **"Community"** in the navigation menu
3. View open issues or click **"New Issue"** to report a problem

## Creating an Issue

To create a new issue:

1. Click **"New Issue"** in the Community section
2. Fill in the issue title and description
3. (Optional) Tag the relevant machine or component
4. Click **"Submit"**

Your team members will be notified and can collaborate to resolve the issue.

## Use Cases

| Use Case | Example |
|:---------|:--------|
| Incident reporting | "Database server disk usage at 95%" |
| Knowledge sharing | "How to rotate vsay-agent API keys" |
| Feature requests | "Need batch command execution across machines" |
| Troubleshooting | "Agent keeps disconnecting after 30 minutes" |

## Community vs Enterprise

The Community feature (issue tracker) is available in both editions:

| Feature | Community | Enterprise |
|:--------|:---------:|:----------:|
| Issue creation and tracking | ✅ | ✅ |
| Team comments and collaboration | ✅ | ✅ |
| Voting on issues | ✅ | ✅ |
| Search | ✅ | ✅ |
